# 钉钉机器人消息队列测试指南

## 测试目标

验证消息队列的以下功能：
1. 消息按顺序处理（FIFO）
2. 一条处理完成才处理下一条
3. 超时保护机制
4. 积压恢复能力

---

## 测试方法

### 方法一：手动测试（推荐）

#### 测试场景1：正常顺序处理

**步骤：**
1. 启动 pi 并连接钉钉机器人
2. 在钉钉中连续发送3条消息：
   - 消息1："测试消息1"
   - 消息2："测试消息2"  
   - 消息3："测试消息3"

**预期结果：**
```
[dingtalkbot] 消息入队 [msg1...] 队列长度: 1
[dingtalkbot] 发送消息给 pi [msg1...]
[dingtalkbot] 已发送给 pi，等待处理完成...

[dingtalkbot] 消息入队 [msg2...] 队列长度: 2
[dingtalkbot] 消息入队 [msg3...] 队列长度: 3

# pi 处理完成后
[dingtalkbot] 消息 msg1... 处理完成，继续下一条
[dingtalkbot] 发送消息给 pi [msg2...]
...
```

**验证点：**
- [ ] 消息按入队顺序处理（msg1 → msg2 → msg3）
- [ ] 处理完一条才发送下一条给 pi
- [ ] pi 会话中不会同时出现多条待处理消息

---

#### 测试场景2：超时保护

**步骤：**
1. 发送一条消息
2. 在 pi 中**不回复**（模拟处理卡住）
3. 等待5分钟
4. 发送第二条消息

**预期结果：**
```
[dingtalkbot] 发送消息给 pi [msg1...]
[dingtalkbot] 已发送给 pi，等待处理完成...

# 5分钟后
[dingtalkbot] 消息 msg1... 处理超时
[dingtalkbot] 消息 msg1... 处理完成，继续下一条
[dingtalkbot] 发送消息给 pi [msg2...]
```

**验证点：**
- [ ] 5分钟后自动超时
- [ ] 超时后继续处理下一条
- [ ] 队列不会永久卡住

---

#### 测试场景3：积压恢复

**步骤：**
1. 断开 pi 连接
2. 在钉钉中连续发送3条消息（消息会入队但无法发送给 pi）
3. 重新连接 pi

**预期结果：**
```
# 断线期间
[dingtalkbot] 消息入队 [msg1...] 队列长度: 1
[dingtalkbot] 消息入队 [msg2...] 队列长度: 2
[dingtalkbot] 消息入队 [msg3...] 队列长度: 3

# 重新连接后
[dingtalkbot] 发送消息给 pi [msg1...]
...
[dingtalkbot] 消息 msg1... 处理完成，继续下一条
[dingtalkbot] 发送消息给 pi [msg2...]
...
```

**验证点：**
- [ ] 断线期间消息入队不丢失
- [ ] 重连后自动继续处理
- [ ] 按顺序处理积压消息

## 常见问题排查

### 问题1：消息没有顺序处理

**现象：** 多条消息同时出现在 pi 会话中

**排查：**
```bash
# 检查日志中是否有以下输出
[dingtalkbot] 发送消息给 pi [msg1...]
[dingtalkbot] 发送消息给 pi [msg2...]  # 错误：应该等待msg1完成

# 正确应该看到
[dingtalkbot] 发送消息给 pi [msg1...]
[dingtalkbot] 消息 msg1... 处理完成，继续下一条
[dingtalkbot] 发送消息给 pi [msg2...]
```

### 问题2：队列卡住不处理

**现象：** 消息入队后没有后续处理

**排查：**
1. 检查 `isProcessing` 状态是否一直为 true
2. 检查 `agent_end` 是否触发
3. 查看是否有超时日志

### 问题3：回复到错误会话

**现象：** pi 回复发送给了错误的钉钉用户

**排查：**
1. 检查 `dingTalkSessions` 中 messageId 是否正确存储
2. 检查 `agent_end` 中是否能正确提取 messageId
3. 检查 `getLatestSession` 是否被错误使用

---

## 关键日志说明

| 日志 | 含义 |
|------|------|
| `消息入队 [xxx...] 队列长度: N` | 新消息加入队列 |
| `发送消息给 pi [xxx...]` | 开始发送给 pi |
| `已发送给 pi，等待处理完成...` | 等待 pi 处理 |
| `消息 xxx... 处理完成` | pi 处理完成（agent_end触发） |
| `消息 xxx... 处理超时` | 超时保护触发 |
| `继续下一条` | 开始处理队列中下一条 |

## 配置文件

如需调整超时时间，修改 `index.ts`：

```typescript
const MESSAGE_TIMEOUT = 5 * 60 * 1000; // 默认5分钟
```

建议生产环境：5分钟  
建议测试环境：30秒
